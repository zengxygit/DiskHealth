💾 DiskHealth

基于 Qt / C++ 的磁盘健康监测工具——实时读取 S.M.A.R.T 数据，评估磁盘健康状态，支持 HDD / SSD / NVMe 全设备类型。

平台：Windows 10/11  |  框架：Qt 5.x / C++17  |  协议：ATA / SCSI / NVMe  |  文档日期：2026-05-09
概述 架构 类图 时序图 SMART属性 信号与槽 业务流程 设备支持 常见问题

✦ 核心功能

  • 磁盘健康评估：综合 SMART 属性判断磁盘处于 Good / Caution / Bad 三级状态
  • 温度监控：实时显示磁盘温度，支持摄氏度 / 华氏度切换，SSD 警戒阈值 60°C，HDD 50°C
  • SMART 属性列表：展示完整属性 ID、名称、当前值、最差值、阈值及 Raw 值
  • 多磁盘切换：ComboBox 快速在多块磁盘间切换查看详情
  • 系统托盘：后台常驻，悬停托盘图标可浮出状态摘要
  • 开机自启：注册表写入自启项，静默后台监测
  • 多语言：内置多语言翻译框架，动态切换界面语言
  • EPM 集成：支持与 EaseUS Partition Master 联动，接收进程间消息
  • 数据上报：可选阿里云日志服务（AliyunLog）记录用户行为
Good — 健康
Caution — 注意
Bad — 危险

📊 S.M.A.R.T 属性详解

ℹ 什么是 S.M.A.R.T？

S.M.A.R.T（Self-Monitoring, Analysis and Reporting Technology，自我监测分析与报告技术）是内置于现代 ATA/SATA/NVMe 硬盘的监控系统，通过一系列属性值反映磁盘的运行状态，可提前预警潜在故障。

健康判断规则：当 CurrentValue ≤ Threshold 时，该属性触发警告，DiskHealth 将根据关键属性（05h / C5h / C6h）的触发情况将磁盘状态从 Good 升级为 Caution 或 Bad。

📋 关键 SMART 属性说明

属性 ID属性名称重要性说明
0x01 (01h)Read Error Rate关键从磁盘表面读取数据时的错误率，值过低表示磁盘表面可能有损伤
0x05 (05h)Reallocated Sectors Count关键已重新分配的扇区数量，即"坏扇区"数，非零值需立即关注。DiskHealth 阈值：1
0x07 (07h)Seek Error Rate重要磁头寻道错误率，高值表示机械部件可能有问题（HDD 专属）
0x09 (09h)Power-On Hours信息通电总小时数，不同厂商单位可能为分钟、半分钟或秒
0x0C (0Ch)Power Cycle Count信息开关机次数，用于评估磁盘使用历史
0xBB (BBh)Reported Uncorrectable Errors关键无法纠正的错误数，非零值表示数据丢失风险极高
0xC2 (C2h)Temperature重要当前温度（°C），SSD 超过 60°C / HDD 超过 50°C 触发警告
0xC5 (C5h)Current Pending Sector Count关键等待重新分配的不稳定扇区数。DiskHealth 阈值：1
0xC6 (C6h)Uncorrectable Sector Count关键无法纠正的扇区数。DiskHealth 阈值：1
0xE7 (E7h)SSD Life Left / Temperature重要SSD 剩余寿命百分比（部分厂商）或温度（东芝 SSD）
0xF1 (F1h)Total Host Writes信息主机总写入量，单位因厂商而异（512B / 1MB / 32MB / GB）
0xF2 (F2h)Total Host Reads信息主机总读取量，用于评估 SSD 使用寿命

🔄 核心业务流程

① 磁盘枚举流程

设置枚举标志 配置 FlagUsbSat / FlagMegaRAID 等标志，决定是否枚举 USB、RAID 等类型设备。
调用 CAtaSmart::Init() 遍历 \\.\PhysicalDrive0 ~ PhysicalDrive55，尝试打开设备句柄。
识别命令类型 根据设备响应确定使用 CMD_TYPE_PHYSICAL_DRIVE、CMD_TYPE_NVME_STORAGE_QUERY、CMD_TYPE_SAT 等命令类型。
读取 SMART 数据 通过 DeviceIoControl（IOCTL_ATA_PASS_THROUGH / DFP_RECEIVE_DRIVE_DATA）或 WMI 读取原始 SMART 字节流。
解析 ATA_SMART_INFO 填充 Model、SerialNumber、Firmware、Temperature、IsSsd、DiskVendorId 等字段。
识别厂商并执行专属解析 根据 Model 字符串匹配 VENDOR_ID，调用对应厂商解析逻辑处理特殊 SMART 属性。
计算健康状态 检查 05h/C5h/C6h/BBh 等关键属性值是否超过 Threshold，综合判断 DISK_STATUS。
通知 UI 更新 RefreshThread::finished → sigDiskInfosChanged → CDiskHealthMainWin::onDiskInfosChanged()。

❓ 常见问题

Q 为何某些 USB 硬盘无法读取 SMART 数据？

USB 外置硬盘需要 USB 桥接芯片支持 SAT（SCSI-ATA Translation）透传。不支持 SAT 的桥接芯片（如某些廉价 USB 3.0 芯片）无法透传 ATA 命令，因此 SMART 数据读取会失败。DiskHealth 支持 JMicron、ASMedia、Realtek、Sunplus、Prolific、Cypress 等主流桥接芯片。

如果 USB 硬盘显示在列表中但 SMART 属性为空，说明桥接芯片不在支持列表中，或需要管理员权限运行程序。

Q 磁盘状态为 Caution，但我感觉硬盘没有问题？

Caution 状态并不意味着硬盘立即会损坏，而是表示一个或多个 SMART 属性已超过警戒值。DiskHealth 对以下属性设置了较低阈值（=1），即使只有 1 个坏扇区也会触发 Caution：

  • 05h Reallocated Sectors Count — 已有磁盘区域被重新分配，说明存在坏扇区
  • C5h Current Pending Sector Count — 存在不稳定扇区，读取时可能出错
  • C6h Uncorrectable Sector Count — 存在无法纠正的错误扇区
建议：即使是 Caution 状态，也应尽快备份重要数据，并持续观察这些属性值的变化趋势。

Q NVMe SSD 的温度和 SMART 属性与 SATA SSD 显示方式有何不同？

NVMe SSD 使用独立的 NVMe SMART/Health Information Log，通过 IOCTL_STORAGE_QUERY_PROPERTY 读取。DiskHealth 通过 NVMeSmartToATASmart() 函数将 NVMe 格式转换为统一的 ATA SMART 属性格式，实现与 SATA 硬盘相同的展示界面。NVMe 支持最多 15 个映射属性（NVME_ATTRIBUTE = 15）。

Q 程序运行需要管理员权限吗？

是的，读取磁盘 SMART 数据需要以 Administrator 身份运行，因为 DeviceIoControl 对 \\.\PhysicalDriveN 的访问需要特权。对于注册系统托盘、访问注册表（注册自启动项）同样需要管理员权限。

建议在 .exe 的清单文件（manifest）中设置 requireAdministrator，或通过右键"以管理员身份运行"启动程序。

Q EPM 模式是什么？

EPM（EaseUS Partition Master）模式是 DiskHealth 作为 EPM 子进程运行时的特殊模式。通过命令行参数 EPM 启动，并通过 QLocalSocket 与父进程通信，接收磁盘切换指令（sigSwitchDiskIndex）并将 SMART 数据发送回 EPM。EPM 模式下，阿里云日志上报会被禁用（inEPM = true）。

Q 如何添加对新 NVMe 厂商的支持？

在 AtaSmart.h 中的 VENDOR_ID 枚举添加新厂商 ID，在 ssdVendorString[] 和 attributeString[] 数组中添加对应字符串，然后在 AtaSmart.cpp 的 Init() 函数中添加 Model 字符串匹配逻辑，最后在 DiskAttr.cpp 的 loadDiskAttrs() 中添加该厂商的属性名称映射表。
